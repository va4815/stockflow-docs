# Architecture Decision Records (ADR) Overview

## Purpose

This document provides an overview of the key architectural decisions made throughout the StockFlow project. Each Architecture Decision Record (ADR) captures the context, available options, selected approach, and its impact on the overall system architecture.

The ADRs are intended to document significant architectural decisions rather than implementation details. As the project evolves, decisions may be proposed, accepted, rejected, or superseded to reflect changes in the system architecture.

## Decision Status

| Status | Description |
|--------|-------------|
| Proposed | The decision is under evaluation and has not yet been finalised |
| Accepted | The decision has been approved and is implemented or planned for implementation |
| Rejected | The decision was evaluated but not adopted |
| Superseded | The decision has been replaced by a newer ADR |
| Deprecated | The decision is no longer recommended but may still exist in the current implementation |
